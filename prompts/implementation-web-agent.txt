## Regras de Implementação

### HTML
- HTML5 semântico: header, nav, main, section, article, aside, footer
- Landmarks ARIA para acessibilidade
- Meta tags completas (viewport, charset, description, og:*)
- Formulários com labels associados, fieldset, legend
- Lazy loading para imagens: loading="lazy"
- Picture element com srcset para responsividade

### CSS
- Mobile-first: min-width breakpoints
- CSS Custom Properties (--color-primary, --spacing-*, --font-*)
- Grid + Flexbox (Grid para layout, Flex para alinhamento)
- BEM naming ou CSS Modules
- Sem !important (exceto reset/normalize)
- Preferir clamp() para tipografia fluida
- Container queries onde suportado
- Animações com transform/opacity apenas (GPU-accelerated)
- Aspect-ratio para media containers
- Dark mode via prefers-color-scheme + toggle

### JavaScript
- ES2022+ features: nullish coalescing, optional chaining, top-level await
- Módulos ES nativos (import/export)
- Fetch API com AbortController para cancelamento
- IntersectionObserver para lazy interactions
- CustomEvent para comunicação entre componentes
- Event delegation para listas dinâmicas
- Debounce/throttle para eventos frequentes
- Template literals para construção de HTML
- Error boundaries: try/catch em async, window.onerror global
- Sem innerHTML quando há user input — usar textContent ou DOM API

### Acessibilidade (WCAG 2.1 AA)
- Contraste mínimo 4.5:1 (texto), 3:1 (componentes)
- Navegação completa por teclado (Tab, Enter, Escape, Arrow keys)
- Focus visible com outline customizado
- Skip links para conteúdo principal
- Aria-live para conteúdo dinâmico
- Role, aria-label, aria-describedby onde necessário
- Reduzir motion: prefers-reduced-motion

### Performance
- Minificação de CSS/JS
- Critical CSS inline no <head>
- Defer/async para scripts não críticos
- Preload para recursos críticos (fonts, hero image)
- Service Worker para cache offline (se PWA)
- Code splitting com dynamic import()
