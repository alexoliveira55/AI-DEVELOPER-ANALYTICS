## Regras de Implementação

### Arquitetura
- Classes orientadas a objeto (DEFINE CLASS ... AS ... OLEPUBLIC)
- Framework MVC com classes PRG (Model/View/Controller separados)
- Data tier com SQL Pass-through para backend SQL Server
- Business objects encapsulando lógica de negócio
- Session classes para isolamento de data sessions

### Código
- LPARAMETERS tipados com AS clause (LPARAMETERS tcNome AS String)
- TRY/CATCH/FINALLY para TODOS os blocos críticos
- LOCAL para todas as variáveis (sem PUBLICs)
- Naming convention: tc=char param, tn=num param, tl=logical param, to=object param
- Constantes via #DEFINE em header .h files
- Assert para validação em dev (SET ASSERTS ON)
- CREATEOBJECT() / NEWOBJECT() para instanciação

### Data Access
- SQL Pass-through: SQLEXEC(), SQLPREPARE(), SQLCONNECT()
- Parameterized queries SEMPRE (? placeholders)
- CURSORSETPROP("Buffering", 5) — optimistic table buffering
- TABLEUPDATE(.T., .T.) com rollback em CATCH
- BEGIN/END TRANSACTION para operações compostas
- Use CursorAdapter para data binding
- SQLSETPROP para timeout e packet size

### UI/Forms
- Form classes herdando de base form (frmBase)
- Grid com colunas configuráveis e headers localizados
- Pageframe para formulários complexos
- Input validation no LostFocus + botão Save
- Mensagens via MESSAGEBOX() com constantes (#DEFINE MB_*)
- Toolbar unificada com navegação CRUD

### Relatórios
- Report Designer (.frx) referenciado via DO REPORT FORM
- Cursor temporário como data source
- Variáveis de relatório para totalizadores
- Preview antes de impressão

### Testes
- FoxUnit para unit tests
- Classe TestCase base com Setup/TearDown
- Assertions: This.AssertEquals, This.AssertTrue, This.AssertNotNull
- Mock objects para data access layer
- Test data isolation com data sessions privadas

### Migração .NET
- Documentar interfaces públicas para futura migration
- Usar COM para interop onde possível
- SQL Server como backend (não DBF para novos projetos)
- Separar lógica de negócio da UI
- wwDotNetBridge para chamar assemblies .NET

### Segurança
- Parameterized queries (NUNCA concatenar strings SQL)
- Validação de input em todas as boundaries
- Encryption para dados sensíveis (Chilkat ou wwEncryption)
- Access control por nível de usuário
- Audit trail para operações CRUD
