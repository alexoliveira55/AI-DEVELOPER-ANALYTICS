## Regras de Implementação

### Arquitetura
- Clean Architecture: Domain → Application → Infrastructure → Presentation
- CQRS com MediatR (Commands separados de Queries)
- Repository Pattern com Unit of Work
- Domain Entities ricas (não anêmicas) com validação interna
- Value Objects para conceitos de domínio (Email, CPF, Money)
- Domain Events para comunicação entre aggregates

### Código
- C# 12+ features: primary constructors, collection expressions, raw string literals
- Nullable reference types HABILITADO — sem `null!` desnecessários
- async/await em toda stack de IO (controllers → services → repositories)
- Extension methods para cross-cutting concerns
- Records para DTOs e Value Objects imutáveis
- Pattern matching para controle de fluxo complexo
- IOptions<T> para configuração tipada

### Entity Framework Core
- Fluent API para configuração de entidades (não Data Annotations)
- Migrations com Up/Down implementados
- Global Query Filters (soft delete, multi-tenant)
- Owned Types para Value Objects
- DbContext com SaveChangesAsync + transaction handling

### Validação & Error Handling
- FluentValidation para input validation
- Result Pattern (não exceções para fluxo de negócio)
- ProblemDetails para API error responses (RFC 7807)
- Global exception handler middleware
- Domain exceptions para regras de negócio violadas

### DI & Middleware
- Registrar todos os services no DI container (extension method por layer)
- Middleware: ExceptionHandler → Authentication → Authorization → Logging → Endpoint
- Scrutor para auto-registro por convenção (se aplicável)

### Testes
- xUnit + Moq + FluentAssertions
- Unit tests para Domain + Application layers
- Arrange-Act-Assert pattern
- Test builders/factories para entidades complexas
- InMemory DbContext para testes de repositório
