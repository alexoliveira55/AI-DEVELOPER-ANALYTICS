## Regras de Implementação

### Princípios
- SOLID — Single Responsibility, Open/Closed, Liskov, Interface Segregation, Dependency Inversion
- DRY — Don't Repeat Yourself
- KISS — Keep It Simple, Stupid
- YAGNI — You Aren't Gonna Need It
- Separation of Concerns — camadas bem definidas
- Favor composition over inheritance
- Program to interfaces, not implementations

### Padrões Obrigatórios
- Repository Pattern para data access
- Service Layer para lógica de negócio
- DTO/VO para transferência de dados entre camadas
- Factory para criação de objetos complexos
- Observer/Event para comunicação desacoplada
- Strategy para algoritmos intercambiáveis

### Código
- Nomes descritivos (sem abreviações obscuras)
- Funções curtas (< 30 linhas idealmente)
- Um nível de abstração por função
- Early returns para reduzir nesting
- Tratamento de erros explícito em todas as boundaries
- Logging estruturado (não print/console.log)
- Configurações externalizadas (não hardcoded)

### Data Access
- Queries parametrizadas SEMPRE (nunca concatenar input do usuário)
- Connection pooling
- Transações para operações compostas
- Migrations versionadas para schema changes
- Índices para campos frequentemente consultados

### Testes
- Unit tests para lógica de negócio
- Integration tests para data access
- Naming: should_expectedBehavior_when_condition
- Arrange/Act/Assert (AAA) pattern
- Mocks/Stubs para dependências externas
- Cobertura mínima: camada de serviço 80%+

### Estrutura de Arquivos
```
src/
  ├── models/         # Entidades de domínio
  ├── repositories/   # Data access layer
  ├── services/       # Business logic
  ├── controllers/    # Entry points (API/CLI/UI)
  ├── dtos/           # Data transfer objects
  ├── validators/     # Input validation
  ├── utils/          # Helpers genéricos
  ├── config/         # Configurações
  └── exceptions/     # Custom error types
tests/
  ├── unit/
  │   ├── services/
  │   └── validators/
  └── integration/
      └── repositories/
```

### Segurança
- Validação de input em todas as boundaries do sistema
- Queries parametrizadas (SQL injection prevention)
- Output encoding (XSS prevention)
- Autenticação e autorização em todas as rotas protegidas
- Secrets em variáveis de ambiente (nunca no código)
- Princípio do menor privilégio

### Documentação
- README com setup, build e run instructions
- Comentários APENAS para "porquê", não "o quê"
- Changelog para mudanças significativas
