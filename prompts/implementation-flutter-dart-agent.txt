## Regras de Implementação

### Arquitetura
- Clean Architecture: Data → Domain → Presentation
- BLoC/Cubit para state management (flutter_bloc)
- Repository Pattern para data access
- Use Cases para lógica de domínio (1 use case = 1 operação)
- Dependency Injection com get_it + injectable
- Either<Failure, Success> para error handling (dartz/fpdart)

### Código
- Null safety ESTRITO — sem `!` desnecessários
- Immutable models com freezed + json_serializable
- Extension methods para utils reutilizáveis
- Typedef para callbacks complexos
- Sealed classes para estados (State, Event, Failure)
- const constructors onde possível
- Streams: preferir BLoC events sobre raw StreamController

### UI/Widgets
- Widget composition (small, focused widgets)
- Responsive com LayoutBuilder + MediaQuery
- Temas centralizados (ThemeData, TextTheme, ColorScheme)
- Adaptative widgets (Material + Cupertino via platform check)
- Custom painters apenas quando necessário
- ListView.builder para listas longas (lazy rendering)

### Data Layer
- Dio para HTTP (interceptors para auth, retry, logging)
- Hive/SharedPreferences para cache local
- Modelos separados: Entity (domain) → Model (data, com fromJson/toJson)
- Error handling: try/catch → mapear para Failure types

### Testes
- Unit tests: Bloc tests com blocTest(), Use Case tests
- Widget tests: pumpWidget + finder + interaction
- Integration tests: integration_test package
- Mocks com mockito (build_runner generate)
- Golden tests para UI crítica
