## Regras de Implementação

### Arquitetura
- FastAPI: routers + dependency injection + middleware
- Django: apps + views + serializers + managers
- Separação clara: routes → services → repositories → models
- Repository pattern para acesso a dados
- Factory pattern para criação de objetos complexos

### Código
- Type hints em TODAS as funções (parâmetros e retorno)
- Pydantic v2 para models/DTOs (model_validator, field_validator)
- async/await para IO-bound operations (FastAPI)
- Dataclasses ou attrs para domain entities
- Context managers para resources (DB connections, files)
- Comprehensions quando melhoram legibilidade (não aninhar 2+)
- f-strings para formatação
- Enums para constantes de domínio
- ABC para interfaces/contratos

### ORM & Database
- SQLAlchemy 2.0 (select style) ou Django ORM
- Alembic migrations com upgrade/downgrade
- Sessões com context manager (with Session() as session:)
- Queries otimizadas: selectinload/joinedload para N+1
- Modelos com __tablename__, __repr__, validação

### Error Handling
- Custom exceptions para domínio (FeatureNotFoundError, ValidationError)
- HTTPException com status codes corretos
- Middleware de error handling global
- Logging estruturado com structlog
- Nunca capturar Exception genérico silenciosamente

### Testes
- pytest + pytest-asyncio + factory-boy + httpx (AsyncClient)
- Fixtures com escopo adequado (function, module, session)
- @pytest.mark.parametrize para variações
- conftest.py com fixtures compartilhados
- Mock de external services com unittest.mock ou respx
- Cobertura mínima de caminhos críticos

### Estrutura de Arquivos
```
feature_name/
  ├── __init__.py
  ├── router.py                # FastAPI router / Django urls
  ├── service.py               # Business logic
  ├── repository.py            # Data access
  ├── models.py                # SQLAlchemy/Django models
  ├── schemas.py               # Pydantic DTOs
  ├── exceptions.py            # Custom exceptions
  ├── dependencies.py          # FastAPI dependencies / DI
  ├── constants.py             # Enums, constantes
  └── tests/
      ├── __init__.py
      ├── conftest.py
      ├── test_service.py
      ├── test_repository.py
      └── test_router.py
```

### Segurança
- Parametrized queries sempre (nunca string interpolation em SQL)
- Input validation via Pydantic (strip, max_length, regex)
- CORS configurado explicitamente
- JWT/OAuth2 com python-jose ou authlib
- Secrets via environment variables (python-decouple ou pydantic-settings)
- Rate limiting com slowapi
